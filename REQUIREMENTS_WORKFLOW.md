# 需求助手与后续开发使用说明

本文档说明如何使用 AgentHub 的需求 AI 助手沉淀需求，并把需求文档接入后续开发流程。

## 1. 创建和维护需求文档

1. 打开 AgentHub，选择工作目录。
2. 进入“需求”页面，点击“新建需求”。
3. 在左侧需求文档中写入背景、目标、范围、验收标准和限制条件。
4. 点击需求 AI，围绕当前文档继续提问，例如：
   - “帮我找出这个需求缺失的问题。”
   - “把这段描述整理成可验收条目。”
   - “检查是否有边界场景遗漏。”
5. AI 返回可写入文档的内容时，先预览变更，再点击“Apply to document”写回需求文档。

需求 AI 对话历史会按“工作目录 + 需求草稿”保存。关闭面板、切换页面或重新打开应用后，再进入同一个需求草稿，历史会自动恢复，可以在之前讨论的基础上继续追问。

## 2. 把需求文档变成开发计划

1. 在需求编辑器中确认文档已保存。
2. 点击“下一步”或生成计划入口。
3. 需求 AI 会读取当前需求文档和已解析的需求块，生成 Markdown 开发计划。
4. 检查计划是否覆盖主要验收标准。
5. 点击“Sync to Todo”，把计划中的 `- [ ]` 清单同步到当前会话 Todo。

建议每个 Todo 写清楚关联需求编号或验收点，便于后续验证。

## 3. 把 Todo 派发给 Agent

1. 回到会话或工作台。
2. 选择要执行的 Todo。
3. 按任务类型选择指定 Agent，或使用调度模式自动分配。
4. 执行过程中保留运行日志、提交记录、文件变更和测试结果。

后续验收会读取需求文档、计划 Todo、运行事件和实现证据，因此开发时不要只在聊天里说明结果，要让关键产物落到代码、测试或运行记录中。

## 4. 用需求文档做实现验收

1. 完成开发任务后，回到对应需求文档。
2. 点击验证入口。
3. 需求 AI 会结合当前需求、Todo 状态、运行事件和实现证据，输出每条验收标准的结果。
4. 只把通过的验收结果写回需求文档；失败或未知项应继续修复后再验证。

这形成闭环：需求文档定义目标，计划拆成 Todo，Agent 执行，验证结果再回写需求文档。

## 5. 推荐规范

- 每个需求文档只描述一个明确功能或一组强相关改动。
- 需求中尽量包含“必须做到”和“不做什么”。
- 验收标准使用可判断的句子，避免只写抽象目标。
- 对 AI 生成内容先预览再写回，避免旧回复覆盖新文档。
- 如果需求文档已被手动改动，基于旧 AI 回复的写回会被拒绝，需要重新询问 AI。
