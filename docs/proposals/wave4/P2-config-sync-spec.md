# P2 多机配置同步 — 规格草案

**代号：** `W4-SYNC`  
**状态：** 草案  

## 1. 目标

让用户在第二台机器恢复 AgentHub 关键配置，**不落盘明文 API Key**。

## 2. 分期

| 期 | 能力 |
|----|------|
| MVP | 加密导出包 + 导入恢复（手动拷贝文件） |
| 二期 | 用户选定同步目录 / WebDAV 适配器 |
| 三期 | 冲突 UI 与字段级合并 |

## 3. 导出包内容（建议）

- `providers.config.v1`：**保持加密形态**（与 backup 策略一致）  
- 外观 / 语言 / 部分 UI 偏好  
- 工作区列表（路径可能本机无效，导入时标记「需重选」）  
- **不包含**：内存库全文、任意仓库文件、PTY 历史  

## 4. 安全要求

- 导出二次口令加密（或 OS keychain 包装）  
- 导入前预览将覆盖的 keys  
- 导入失败不破坏现有 config（写 tmp + rename）  

## 5. 与现有备份关系

| | 本地 Backup（Wave2 F-W8） | P2 同步包 |
|--|---------------------------|-----------|
| 场景 | 本机回滚 | 跨机迁移 |
| 加密 | store 原样 | 再包一层用户口令 |
| UI | 设置 → 配置备份 | 设置 → 同步 / 迁移 |

## 6. 验收

- 机器 A 导出 → 机器 B 导入 → 至少一个 provider 可用（重填 key 策略需文档说明）  
- 错误口令无法解密
